# Meta API連携コスト試算書

## 前提条件

### Meta Marketing APIの基本情報
- **API利用料**: **無料**（Meta Marketing APIは基本的に無料で利用可能）
- **レート制限**: アプリごとに異なる（通常、1時間あたり200-600リクエスト）
- **データ取得**: 広告アカウント、キャンペーン、広告セット、広告のパフォーマンスデータを取得可能

### 想定する連携方式
1. **n8n（自己ホスト）を使用した自動データ取得**
   - 無料（サーバー管理が必要）
   - 1日1回、自動でMeta APIからデータを取得
   - 取得したデータを本システムのAPI経由でアップロード

2. **Make（旧Integromat）を使用した自動データ取得**
   - クラウドサービス（サーバー管理不要）
   - 1日1回、自動でMeta APIからデータを取得
   - 取得したデータを本システムのAPI経由でアップロード

3. **直接API連携（本システムから直接Meta APIを呼び出し）**
   - 本システムのバックエンドから直接Meta APIを呼び出し
   - ユーザー認証（OAuth）が必要

## 注意事項

1. **Meta APIのレート制限**
   - アプリごとにレート制限が異なる
   - 大量のユーザーがいる場合、レート制限に達する可能性がある
   - 対策: リクエストを分散させる、キャッシュを活用する

2. **トークンの有効期限**
   - Meta APIのトークンは60日で期限切れ
   - 自動リフレッシュ機能が必要
   - ユーザーに再認証を求める必要がある場合もある

3. **データ量の増加**
   - キャンペーン数が多い場合、データ量が増加
   - ストレージコストが増加する可能性がある
   - 対策: データの圧縮、古いデータのアーカイブ

4. **エラーハンドリング**
   - API呼び出しが失敗した場合のリトライ機能
   - エラーログの監視
   - ユーザーへの通知機能

## 結論

**Meta API連携のコストは非常に低い**です。主なコストは：
- **初回セットアップ（n8n/Make経由）**: ¥0 - ¥2,500（ユーザー自身が実施する場合は無料）
- **初回セットアップ（直接API連携）**: ¥120,000（1回限り）
- **月額コスト（1ユーザー）**: ¥0.05/月（n8n/Make無料プラン使用時）
- **月額コスト（10ユーザー）**: ¥75 - ¥135/ユーザー（n8n/Make使用時）
- **月額コスト（100ユーザー）**: ¥7.55 - ¥13.55/ユーザー（n8n/Make使用時）
- **月額コスト（1,000ユーザー以上）**: ¥0.83/ユーザー（直接API連携）

**Meta API自体は無料**であり、n8nも自己ホスト版は**完全無料**、Makeも無料プラン（1,000操作/月）があります。コストの大部分は：
1. サーバー/インフラコスト（n8n使用時、新規サーバーが必要な場合のみ）
2. Makeの料金プラン（操作数が多い場合）
3. 既存サーバーを使用する場合は、追加コストはほぼ¥0

**推奨**: 
- **個人利用**: n8n（既存サーバー使用）またはMake（無料プラン）で追加コストほぼ¥0
- **小規模**: n8n経由が最もコスト効率が良い
- **大規模**: 直接API連携が最もコスト効率が良い
